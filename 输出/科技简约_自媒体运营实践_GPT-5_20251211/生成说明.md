## 一、生成工具与模型

- **生成工具**：本地浏览器 + 手工前端开发（HTML / CSS / JavaScript）
- **辅助环境**：现代桌面浏览器（推荐最新版 Chrome / Edge / Safari）
- **设计与文案生成模型**：`GPT-5`（推理与文案辅助），对整体内容结构、布局语言与创意文案进行辅助规划

## 二、生成参数配置

- **场景**：抖音合集封面图（仅表现主题与故事内容，不出现平台名称、尺寸或保存提示等制作信息）
- **尺寸 / 比例**：4:3 比例的自适应画布（使用 `aspect-ratio: 4 / 3`，在浏览器中按视口自动缩放）
- **参考分辨率**：视觉设计参考为约 `1600 × 1200` 像素级别（实际由浏览器渲染像素决定）
- **风格**：科技简约、理性克制、偏冷静的产品级信息排版  
  - 关键词：**高对比、结构化、无渐变、无紫色、内容导向**
- **平台**：抖音短视频合集入口封面（适合在合集列表或个人主页中突出某一运营专题）
- **文案主题**：`自媒体运营实践`
  - 主视觉关键词：`运营`、`实践`
  - 辅助标签：`冷启动`、`选题`、`脚本`、`数据复盘`
  - 故事分段：`起步` / `放大` / `变现` 三个章节

## 三、整体风格总结

- **1）视觉气质**
  - 主基调为 **黑白 + 暖橙点缀**，避免使用紫色与大面积渐变色，整体偏向稳重的产品运营风，而非炫技视觉。
  - 使用大字号的黑底白字块突出“运营”关键词，形成**一眼可识别的视觉锚点**。
  - 通过小范围色块、描边与阴影来提供空间感，而不是依赖渐变背景。

- **2）版式结构**
  - 左侧为“主题 + 关键信息密度区”，右侧为“章节故事卡片区”，采用 **左右分栏 + 不完全对称位移**，增强动感。
  - 使用 3–4 个字号层级：  
    - 最大：`运营 / 实践` 主关键词  
    - 中级：章节标题（如“从 0 到 1 搭建内容阵地”）  
    - 常规：正文说明与标签  
    - 最小：角标与英文辅助信息
  - 通过 `aspect-ratio` 控制整体画布，保证输出为标准 4:3 比例。

- **3）信息节奏**
  - 先读左侧主标题（运营 / 实践）→ 再读“自媒体运营实践 + 一句说明”→ 再由右侧三张卡片讲述故事。
  - 卡片自上而下为“起步 / 放大 / 变现”，与运营路径自然对齐，方便用户在一眼之间建立结构化认知。

## 四、生成思路与实现步骤

### 1. 内容结构设计

1. 提取主题文案 `自媒体运营实践`，拆分出更有冲击力的核心词：**运营 / 实践**。  
2. 将运营路径划分为三个阶段：  
   - **起步**：冷启动与定位  
   - **放大**：把爆款方法变成流程  
   - **变现**：内容到转化的闭环  
3. 为每个阶段写出一句简洁、可视化的说明文案，保持“运营建议”而非“教程说明书”的语气。

### 2. 版式与交互实现

1. 使用 HTML 构建主结构：`#poster` 作为唯一被截取的封面容器，上面不出现任何平台名或尺寸说明；操作区（保存按钮）放在外部容器中。
2. 使用 CSS `aspect-ratio: 4 / 3` 锁定海报比例，通过 `max-width` + `min(100%, 100vh * 4 / 3)` 控制在窗口内自适应缩放。
3. 左侧采用块状标题 + 多层级文字排版，右侧采用三张错位卡片，制造可视化的“阅读路径”和“信息节奏感”。
4. 通过细边框、阴影与小色块（如角落的橙色圆点）提供层次感，避免使用大面积渐变与 AI 风格图标。

### 3. 技术实现细节

1. 使用 `html2canvas`：  
   - 仅对 `#poster` 元素进行截图，保证输出图片中只包含海报内容本身。  
   - 设置 `scale: 2` 提升导出清晰度，满足常见屏幕下的展示需要。
2. 使用 JavaScript 组织交互逻辑：  
   - 提供 `getPosterElement` / `getSaveButtonElement` / `createFileName` 等函数，分别负责 DOM 获取与文件名生成。  
   - `capturePosterAndSave` 函数负责调用 `html2canvas` 并触发下载。  
   - 所有函数均在顶部通过中文注释标注输入与输出，便于后续维护与二次开发。

## 五、灵感来源与“转译实现”说明

> 仅从以下灵感池中采样 2–3 位设计师的“可转译特征”，不临摹具体作品。

本次设计参考的灵感来源以及在网页中的转译方式如下：

- **1）Massimo Vignelli（信息架构与网格思维）**
  - 灵感点：极简主义、严格的网格系统、有限色彩与高度秩序感。
  - 网页转译：
    - 使用清晰的左右分栏网格，左侧信息密度更高、右侧为分段信息，整体布局近似“理性报表式设计”。
    - 控制色彩数量在黑 / 白 / 灰 + 橙色点缀的范围内，形成高度可控的视觉系统。

- **2）Paula Scher（文字作为主视觉）**
  - 灵感点：将文字本身当作图形元素，利用大字号与对比形成强烈视觉焦点。
  - 网页转译：
    - 将“运营 / 实践”两个词以大字号与块状背景呈现，成为画面主角，而非依赖插画或图像。
    - 辅助文案保持简洁，更多作为“节奏”而非堆砌说明。

- **3）Saul Bass（极简图形与强烈对比）**
  - 灵感点：鲜明的对比色块、几何形态、用少量元素讲清故事。
  - 网页转译：
    - 通过三张错位的“故事卡片”，用极简信息讲述起步 / 放大 / 变现三阶段，结构与对比优先于装饰。
    - 使用少量橙色点缀（如角落圆点与下划线），在整体克制的基础上制造记忆点。

通过上述方式，保证了**灵感仅停留在可转译的设计特征层面**，未复用任何具体作品的版式、配色组合、字体搭配或品牌元素。
